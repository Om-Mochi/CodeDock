
<h6>CodeDock</h6>

![CodeDock](CodeDock/icons/github_demo/win_demo.png)

CodeDock is an advanced code editor and docking framework built with Python and PyQt6. The project was created to explore desktop application development, custom UI frameworks, software architecture, editor infrastructure, and Language Server Protocol (LSP) integration.

«⚠️ This project is primarily a learning project and is not intended to represent production-quality architecture or code organization.»

---

Features

Advanced Docking System

![ADSDemo](CodeDock/icons/ADS_Demo.gif)

- Dynamic dock placement
- Drag-and-drop docking
- Tabbed dock containers
- Nested splitter layouts
- Custom docking architecture

Code Editing

- Syntax highlighting
- Theme customization
- Code navigation
- Reference lookup
- Multi-language support

Language Server Protocol (LSP)

![AutoComplition](CodeDock/icons/cd_auto_completion.gif)

- Python (pylsp)
- C/C++ (clangd)
- Rust (rust-analyzer)
- Diagnostics
- Hover information
- Signature help
- Auto-completion

Editor Infrastructure

- Custom minimap

![AutoComplition](CodeDock/icons/cd_mini_map.gif)

- Breakpoints
- Workspace management
- Tab switcher
- Reference viewer
- Multi-threaded architecture

What I Learned

CodeDock became one of my largest personal projects and taught me many important lessons about software development:

- Desktop application architecture
- Custom widget development
- Event-driven systems
- Multi-threaded programming
- Language Server Protocol integration
- UI framework design
- Project organization and maintainability

As the project grew, the architecture became increasingly difficult to maintain. Looking back, there are many design decisions I would approach differently today. The codebase contains experimental ideas, inconsistent patterns, and areas that would benefit from significant refactoring.

However, those challenges became valuable learning experiences and helped me better understand scalable software design.

Future Plans

I am currently rebuilding many of the concepts from CodeDock in a new C++/Qt project.

The goals of the new implementation are:

- Cleaner architecture
- Better code organization

- Stronger separation of concerns
- Better performance
- Modern C++ design practices

CodeDock remains an important milestone in my learning journey and serves as the foundation for future projects.
